# Contributing

Contributions are welcome – keep it lean and predictable.

## Scope
- ESPHome YAML
- Home Assistant configuration
- Fokus: Stabilität, Wartbarkeit, kein Overengineering

## How to contribute
1. Fork the repo
2. Create a feature or fix branch
3. Make your changes
4. Open a Pull Request

## Rules of engagement
- One topic per PR
- No breaking changes without explanation
- Keep configs readable (comments > cleverness)
- Stick to ESPHome / HA best practices
- Test before submitting (at least syntactically)

## What won’t be accepted
- Untested configs
- Vendor-specific hardcoding without reason
- Large refactors without discussion

## Style
- Clear naming
- Minimal magic
- YAML > Templates, unless justified
